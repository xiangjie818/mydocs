Ceph集群容灾域调整

默认配置下，Ceph集群的容灾域是host，对于节点比较多的集群，建议使用rack作为容灾域。以rack作为容灾域时，同一个rack内的节点全部故障时，Ceph集群仍然可以工作。

修改集群容灾域为rack，步骤如下：

  1. 创建rack域

  2. 把三个新建的rack，加入default下

  3. 把节点加入rack域

  4. 获取集群的map信息

  5. 对map信息进行反编译

  6. 修改容灾域
    把monmap.txt的rule配置中的step chooseleaf firstn 0 type host，替换为step chooseleaf firstn 0 type rack

  7. 编译生成系统能识别的map

  8. 将map信息注入到集群

  9. 验证
